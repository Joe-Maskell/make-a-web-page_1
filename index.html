Lesson 2: HTML with structure

Developer tools

In the second lesson, I learnt how to inspect the code used to create webpages by using the developer tools in Google Chrome.

Boxes

I leant that a webpage is composed of boxes and that it is necessary to plan out how these boxes will appear on the page.

CSS

I learnt about the HTML tags that can be used to structure a page and how CSS can then be used to add style to the page

Cascading

There is a heirachy of styles in CSS that are passed on from 'parent' to 'child' unless explicitly altered (hence CASCADING Style Sheets).

Lesson 3: Adding CSS Style to HTML Structure

Summary

In the third lesson, I learnt more about using the developer tools in Google Chrome. I was introduced to the way CSS can be used to position and format the boxes that make up a webpage. Key to this is the ability to temporary change elements using the developer tools.

Checking

I learnt that HTLM and CSS code can be checked for syntax errors using sites on the internet.

Repetition

I learnt that a key principal in programming is not to repeat code. NOT to repeat code.
This is because repeating code produces more code and so more work. If you need to make a change to the code then you have to make it in multiple places. This is more work and makes code less managable and much more prone to error.
Do not repeat code.
